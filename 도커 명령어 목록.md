## 📚 도커 명령어 분야별 모음 목록 (핵심 정리)

IT 전문가로서 도커를 효율적으로 사용하기 위해 필수적인 명령어들을 **객체(Object)별** 및 **기능별**로 분류하여 핵심만 정리해 드립니다. 컨테이너 관리, 이미지 빌드, 네트워크 설정 등 분야별로 빠르게 참조하실 수 있습니다.

---

### 1. 🐳 컨테이너 관리 명령어 모음

컨테이너의 생명주기(Lifecycle)를 제어하고 상호작용하는 데 사용되는 가장 기본적인 명령어들입니다.

| 명령어 | 형식 (축약 가능) | 역할 |
| :--- | :--- | :--- |
| **실행** | `docker run <이미지> <명령>` | 이미지로부터 **새 컨테이너를 생성하고 실행** |
| **목록** | `docker container ls` | **실행 중인** 컨테이너 목록 확인 (`docker ps`로 축약 가능) |
| **전체 목록** | `docker container ls -a` | **모든** 컨테이너 목록 (실행/종료 포함) 확인 (`docker ps -a`로 축약 가능) |
| **시작** | `docker start <컨테이너>` | 종료된 컨테이너 **재실행** |
| **중지** | `docker stop <컨테이너>` | 실행 중인 컨테이너 **정상 종료** |
| **강제 종료** | `docker kill <컨테이너>` | 실행 중인 컨테이너 **강제 종료** |
| **제거** | `docker container rm <컨테이너>` | 컨테이너 **삭제** |
| **로그** | `docker container logs <컨테이너>` | 컨테이너의 **표준 출력/에러 로그** 확인 |
| **내부 진입** | `docker exec -it <컨테이너> <쉘>` | 실행 중인 컨테이너 내부에 **새로운 프로세스 실행** (가장 안전한 접근법) |
| **연결** | `docker attach <컨테이너>` | 실행 중인 컨테이너의 **메인 프로세스**에 I/O 스트림 연결 (주의 필요) |
| **상태** | `docker container inspect <컨테이너>` | 컨테이너의 **로우 레벨 메타데이터** (JSON) 확인 |

### 2. 🖼️ 도커 이미지 명령어 모음

이미지를 관리하고, 레지스트리에서 가져오거나 내보내는 데 사용되는 명령어들입니다.

| 명령어 | 형식 | 역할 |
| :--- | :--- | :--- |
| **목록** | `docker image ls` | **로컬 저장소**의 이미지 목록 확인 |
| **가져오기 (Pull)** | `docker image pull <이미지:태그>` | **원격 레지스트리**에서 이미지 다운로드 |
| **빌드** | `docker image build -t <이름> .` | `Dockerfile`을 사용하여 **새 이미지 생성** |
| **태그 지정** | `docker image tag <소스> <대상>` | 이미지에 새로운 **이름(태그)** 지정 |
| **레지스트리 전송** | `docker image push <이미지:태그>` | 로컬 이미지를 **원격 레지스트리**에 업로드 |
| **히스토리** | `docker image history <이미지>` | 이미지를 구성하는 **레이어 구조** 및 빌드 과정 확인 |
| **제거** | `docker image rm <이미지>` | 로컬 저장소에서 **이미지 삭제** |
| **상세 정보** | `docker image inspect <이미지>` | 이미지의 **로우 레벨 메타데이터** (JSON) 확인 |

### 3. 🌐 도커 네트워크 명령어 모음

컨테이너 간 통신 및 호스트와의 연결을 설정하고 관리하는 명령어들입니다.

| 명령어 | 형식 | 역할 |
| :--- | :--- | :--- |
| **목록** | `docker network ls` | **도커 네트워크** 목록 확인 |
| **생성** | `docker network create <이름>` | **새로운 네트워크** 생성 (컨테이너 격리 및 연결에 사용) |
| **삭제** | `docker network rm <네트워크>` | 네트워크 **삭제** |
| **연결** | `docker network connect <네트워크> <컨테이너>` | 실행 중인 컨테이너를 **네트워크에 연결** |
| **연결 해제** | `docker network disconnect <네트워크> <컨테이너>` | 컨테이너를 **네트워크에서 해제** |
| **상세 정보** | `docker network inspect <네트워크>` | 네트워크의 **상세 정보** (IP 범위, 연결된 컨테이너 등) 확인 |

### 4. 🗄️ 볼륨 및 시스템 명령어 모음 (기타 필수)

데이터 영속성 및 도커 환경 관리에 필요한 명령어들입니다.

| 명령어 | 형식 | 역할 |
| :--- | :--- | :--- |
| **볼륨 목록** | `docker volume ls` | **영구 저장소 볼륨** 목록 확인 |
| **볼륨 생성** | `docker volume create <이름>` | **새 볼륨 생성** |
| **볼륨 제거** | `docker volume rm <볼륨>` | 볼륨 **삭제** (데이터도 함께 삭제됨) |
| **상태 확인** | `docker info` | 도커 **데몬 및 시스템 전반**의 정보 확인 |
| **공간 정리** | `docker system prune` | **사용되지 않는** 이미지, 컨테이너, 네트워크, 캐시 등을 **일괄 삭제**하여 디스크 공간 확보 |

이 명령어 목록을 통해 도커 환경을 전문적으로 관리하는 데 필요한 모든 기본 도구를 갖추셨습니다.
