## 📋 도커 명령어 분야별 완전 목록 (Total Command Reference)

 도커 환경을 전문적으로 운영하는 데 필요한 모든 핵심 명령어들을 **관리 객체별**로 분류하고, 사용 빈도나 중요도와 관계없이 **가능한 한 모두** 추가하여 상세한 목록을 제공합니다.

---

### 1. 🐳 컨테이너 (Container) 관리 명령어

| 명령어 | 형식 | 역할 | 추가/상세 설명 |
| :--- | :--- | :--- | :--- |
| **run** | `docker run [OPTIONS] IMAGE [COMMAND]` | 이미지로부터 **새 컨테이너를 생성하고 실행** | `-d` (백그라운드), `-p` (포트 매핑), `-v` (볼륨 마운트) 등 필수 옵션 포함 |
| **start** | `docker start [OPTIONS] CONTAINER [CONTAINER...]` | 종료된 컨테이너 **재실행** | `-a` (재시작과 동시에 attach) 옵션 |
| **stop** | `docker stop [OPTIONS] CONTAINER [CONTAINER...]` | 실행 중인 컨테이너 **정상 종료** | |
| **restart** | `docker restart [OPTIONS] CONTAINER [CONTAINER...]` | 컨테이너 **재시작** | |
| **kill** | `docker kill [OPTIONS] CONTAINER [CONTAINER...]` | 컨테이너 **강제 종료** (SIGKILL 시그널) | |
| **pause** | `docker pause CONTAINER [CONTAINER...]` | 실행 중인 컨테이너 내의 **모든 프로세스 일시 중지** | |
| **unpause** | `docker unpause CONTAINER [CONTAINER...]` | 일시 중지된 컨테이너 **재개** | |
| **ls (ps)** | `docker container ls [OPTIONS]` | **실행 중인** 컨테이너 목록 확인 (`docker ps`의 별칭) | `-a` (모두 포함), `-q` (ID만 출력) 옵션 |
| **rm** | `docker container rm [OPTIONS] CONTAINER [...]` | 컨테이너 **삭제** | `-f` (실행 중이어도 강제 삭제), `-v` (관련 볼륨도 삭제) |
| **logs** | `docker container logs [OPTIONS] CONTAINER` | 컨테이너의 **표준 출력/에러 로그** 확인 | `-f` (실시간), `--tail N` (마지막 N줄) 옵션 |
| **exec** | `docker exec [OPTIONS] CONTAINER COMMAND [ARG...]` | 실행 중인 컨테이너 내부에 **새로운 프로세스 실행** | `-it` (상호작용 쉘 접근 시 필수) |
| **attach** | `docker attach [OPTIONS] CONTAINER` | 컨테이너의 **메인 프로세스** I/O 스트림에 연결 | |
| **inspect** | `docker container inspect [OPTIONS] CONTAINER [...]` | 컨테이너의 **로우 레벨 메타데이터** (JSON) 확인 | |
| **top** | `docker container top CONTAINER [OPTIONS]` | 컨테이너 내에서 실행 중인 **프로세스 목록** 확인 | |
| **stats** | `docker container stats [OPTIONS] [CONTAINER...]` | 컨테이너의 **CPU, 메모리, 네트워크** 사용량 실시간 확인 | |
| **prune** | `docker container prune [OPTIONS]` | **종료된** 모든 컨테이너 **일괄 삭제** | |
| **rename** | `docker container rename OLD_NAME NEW_NAME` | 컨테이너 **이름 변경** | |
| **wait** | `docker container wait CONTAINER [CONTAINER...]` | 컨테이너가 **종료될 때까지** 블로킹 | |
| **commit**| `docker commit [OPTIONS] CONTAINER [REPOSITORY[:TAG]]`| 실행 중인 컨테이너의 상태를 **새 이미지로 저장** | **(Bad Practice)** 일반적으로 Dockerfile 빌드를 권장 |

### 2. 🖼️ 도커 이미지 (Image) 관리 명령어

| 명령어 | 형식 | 역할 | 추가/상세 설명 |
| :--- | :--- | :--- | :--- |
| **ls** | `docker image ls [OPTIONS]` | **로컬 저장소**의 이미지 목록 확인 (`docker images`의 별칭) | |
| **pull** | `docker image pull [OPTIONS] NAME[:TAG]` | **원격 레지스트리**에서 이미지 다운로드 | |
| **build** | `docker image build [OPTIONS] PATH | URL` | `Dockerfile`을 사용하여 **새 이미지 생성** | `-t` (태그 지정), `--no-cache` 옵션 |
| **push** | `docker image push NAME[:TAG]` | 로컬 이미지를 **원격 레지스트리**에 업로드 | |
| **rm (rmi)** | `docker image rm [OPTIONS] IMAGE [IMAGE...]` | 로컬 저장소에서 **이미지 삭제** (`docker rmi`의 별칭) | `-f` (강제 삭제) 옵션 |
| **tag** | `docker image tag SOURCE_IMAGE[:TAG] TARGET_IMAGE[:TAG]` | 이미지에 새로운 **이름(태그)** 지정 | |
| **history** | `docker image history [OPTIONS] IMAGE` | 이미지를 구성하는 **레이어 구조** 및 빌드 과정 확인 | |
| **inspect** | `docker image inspect [OPTIONS] IMAGE [IMAGE...]` | 이미지의 **로우 레벨 메타데이터** (JSON) 확인 | |
| **prune** | `docker image prune [OPTIONS]` | **사용되지 않는** 이미지 **일괄 삭제** | `-a` (사용하지 않는 모든 이미지 삭제) 옵션 |
| **load** | `docker image load [OPTIONS]` | 파일(`tar` 아카이브)로부터 이미지 **불러오기** | |
| **save** | `docker image save [OPTIONS] IMAGE [IMAGE...]` | 이미지를 파일(`tar` 아카이브)로 **저장** | |

### 3. 🌐 도커 네트워크 (Network) 명령어 모음

| 명령어 | 형식 | 역할 | 추가/상세 설명 |
| :--- | :--- | :--- | :--- |
| **ls** | `docker network ls [OPTIONS]` | **도커 네트워크** 목록 확인 | |
| **create** | `docker network create [OPTIONS] NETWORK` | **새로운 네트워크** 생성 | `-d` (드라이버 지정: bridge, overlay 등), `--subnet`, `--gateway` 옵션 |
| **rm** | `docker network rm NETWORK [NETWORK...]` | 네트워크 **삭제** | |
| **connect** | `docker network connect [OPTIONS] NETWORK CONTAINER` | 실행 중인 컨테이너를 **네트워크에 연결** | |
| **disconnect**| `docker network disconnect [OPTIONS] NETWORK CONTAINER` | 컨테이너를 **네트워크에서 해제** | |
| **inspect** | `docker network inspect [OPTIONS] NETWORK [NETWORK...]` | 네트워크의 **상세 정보** (IP 범위, 연결된 컨테이너 등) 확인 | |
| **prune** | `docker network prune [OPTIONS]` | **사용되지 않는** 네트워크 **일괄 삭제** | |

| :--- | :--- | :--- | :--- |

### 6. 도커 PS (프로세스) 명령어와 옵션들

Docker 컨테이너를 관리할 때 가장 기본이 되는 `docker ps` 명령어와 그 주요 옵션들을 정리했습니다. `docker ps`는 현재 실행 중인 컨테이너의 상태를 확인하는 **프로세스 상태(Process Status)** 명령어입니다.

---

## 1. 기본 명령어 및 주요 옵션

| 명령어 | 설명 | 비고 |
| --- | --- | --- |
| `docker ps` | **실행 중인** 컨테이너 목록만 확인 | 기본값 |
| `docker ps -a` | **모든** 컨테이너 목록 확인 (종료된 컨테이너 포함) | `--all` |
| `docker ps -q` | 컨테이너의 **ID**만 출력 | `--quiet` (스크립트 활용 시 유용) |
| `docker ps -l` | **마지막**으로 생성된 컨테이너 확인 | `--latest` |
| `docker ps -n 3` | 최근 생성된 컨테이너 **n개** 확인 | 예: `-n 3`은 최근 3개 |
| `docker ps -s` | 컨테이너의 **파일 크기**를 함께 표시 | `--size` |

## 2. 실전에서 유용한 필터링 및 포맷팅

단순히 목록을 보는 것 외에, 특정 조건의 컨테이너를 찾거나 보고 싶은 정보만 골라볼 때 유용합니다.

### 컨테이너 필터링 (`--filter` 또는 `-f`)

특정 상태나 이름을 가진 컨테이너만 골라낼 수 있습니다.

* **상태별 필터링:** `docker ps -f "status=exited"` (종료된 컨테이너만)
* **이름별 필터링:** `docker ps -f "name=my-db"`
* **이미지별 필터링:** `docker ps -f "ancestor=nginx"` (nginx 이미지로 만든 컨테이너)

### 출력 형식 지정 (`--format`)

원하는 컬럼만 선택해서 깔끔하게 보고 싶을 때 사용합니다. [Go 템플릿](https://www.google.com/search?q=https://docs.docker.com/reference/cli/docker/container/ls/%23format) 형식을 따릅니다.

* **예시:** ID, 이미지명, 상태만 표 형태로 출력
```bash
docker ps --format "table {{.ID}}\t{{.Image}}\t{{.Status}}"

```

## 3. 자주 하는 실수 및 실전 팁

> **💡 실전 팁 (Best Practice)**
> * **컨테이너가 안 보여요!**: `docker ps`를 입력했는데 아무것도 안 나온다면 컨테이너가 생성되지 않았거나, 실행 직후 오류로 **종료(Exited)**된 상태일 확률이 높습니다. 이럴 땐 반드시 `docker ps -a`로 전체 상태를 확인하세요.
> * **일괄 삭제 팁**: 종료된 모든 컨테이너를 한 번에 지우고 싶을 때 `docker rm $(docker ps -a -q -f status=exited)` 조합을 자주 사용합니다. (또는 최신 버전에서는 `docker container prune` 권장)

## 4. 출력 항목(Column)의 의미

`docker ps` 실행 시 상단에 표시되는 항목의 의미는 다음과 같습니다.

* **CONTAINER ID**: 컨테이너의 고유 ID (앞 12자리만 표시)
* **IMAGE**: 컨테이너를 생성할 때 사용한 이미지명
* **COMMAND**: 컨테이너가 시작될 때 실행되는 명령어
* **CREATED**: 컨테이너가 생성된 시점
* **STATUS**: 현재 상태 (Up: 실행 중, Exited: 종료, Paused: 일시정지)
* **PORTS**: 호스트와 연결된 포트 정보
* **NAMES**: 컨테이너에 설정된 이름 (설정 안 할 시 임의의 이름 부여)
