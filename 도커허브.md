## 🐳 Docker Hub 등록, 이미지 푸시(Push), 그리고 다운로드 방법

Docker Hub는 Docker 이미지를 저장하고 공유하는 **공개 레지스트리 서비스**입니다. 개발자들이 이미지를 손쉽게 관리하고, 전 세계와 협업할 수 있도록 돕는 핵심적인 인프라입니다.

IT 전문가로서 Ubuntu 환경을 기준으로, Docker Hub 계정 생성부터 이미지 등록 및 다운로드까지의 전 과정을 실습 예제 중심으로 단계별로 안내합니다.

-----

### 1\. 📝 Docker Hub 계정 등록 (Sign Up)

Docker Hub에 이미지를 등록(Push)하거나 개인 저장소에 이미지를 다운로드(Pull)하려면 계정이 필수입니다.

1.  **웹사이트 접속:** [Docker Hub 웹사이트](https://hub.docker.com)에 접속합니다.
2.  **계정 생성:** \*\*'Sign Up'\*\*을 클릭하고 사용자 이름(Username), 이메일, 비밀번호 등을 입력하여 계정을 생성합니다.
      * **사용자 이름(Username)이 중요합니다.** 이 이름이 나중에 이미지 경로(`[Username]/[Image Name]`)의 접두사로 사용됩니다.

### 2\. 🔑 Docker Hub 로그인 (Login)

Ubuntu 터미널에서 로컬 Docker 클라이언트가 Docker Hub에 접속하고 인증할 수 있도록 로그인합니다.

```bash
docker login
```

  * **프롬프트:** 사용자 이름(Username)과 비밀번호(Password)를 순서대로 입력합니다.
  * **성공 메시지:** `Login Succeeded` 메시지가 출력되면 성공입니다.
  * **보안 팁:** 비밀번호가 텍스트 파일로 저장되지 않도록, Docker는 기본적으로 \*\*자격 증명 도우미(Credential Helper)\*\*를 사용합니다.

### 3\. 📤 이미지 등록 (Push) 방법

로컬에서 생성한 Docker 이미지를 Docker Hub의 개인 저장소에 업로드하는 과정입니다.

#### 3.1. 🖼️ 이미지 태그 지정 (Tagging)

Docker Hub에 이미지를 올리려면, 반드시 정해진 형식으로 \*\*태그(Tag)\*\*를 지정해야 합니다.

  * **필수 형식:** `[Docker Hub 사용자 이름]/[이미지 이름]:[태그]`

**예제 실습:** 로컬에 `my_app:v1`이라는 이미지가 있다고 가정하고, 제 사용자 이름이 `expertuser`라고 가정합니다.

```bash
# 로컬 이미지 확인
docker images

# 이미지에 Docker Hub 형식으로 태그 지정
docker tag my_app:v1 expertuser/my_app:v1
```

#### 3.2. 🚀 이미지 푸시 (Push)

태그가 지정된 이미지를 Docker Hub로 업로드합니다.

```bash
# Docker Hub로 이미지 푸시
docker push expertuser/my_app:v1
```

  * **진행:** 이미지의 각 레이어(Layer)가 업로드되는 진행 상황이 표시됩니다.
  * **확인:** 푸시가 완료되면 [Docker Hub 웹사이트](https://hub.docker.com)에 로그인하여 해당 이미지(`expertuser/my_app`)가 성공적으로 등록되었는지 확인할 수 있습니다.

-----

### 4\. 📥 이미지 다운로드 (Pull) 및 사용 방법

Docker Hub에 등록된 이미지를 로컬 시스템으로 가져와 사용하는 방법입니다.

#### 4.1. 🌐 공개 이미지 다운로드 (Pull)

공식 이미지나 누구나 다운로드할 수 있도록 공개된 이미지는 사용자 이름 없이 이미지 이름만 사용합니다. (Docker Hub의 공식 레지스트리 경로가 생략됨)

```bash
# Ubuntu 공식 이미지 다운로드 (Docker Hub에서 가장 많이 사용하는 경우)
docker pull ubuntu:latest

# Nginx 공식 이미지 다운로드
docker pull nginx:stable
```

#### 4.2. 👥 개인 저장소 이미지 다운로드 (Pull)

다른 사람의 저장소에 있는 이미지나 자신이 푸시했던 이미지를 다운로드할 때는 **반드시 사용자 이름을 포함**해야 합니다.

```bash
# 다른 사용자(expertuser)의 저장소에서 이미지 다운로드
docker pull expertuser/my_app:v1
```

#### 4.3. 🏃 이미지 실행 (Run)

다운로드한 이미지를 사용하여 컨테이너를 실행합니다.

```bash
# 다운로드한 이미지를 기반으로 컨테이너 실행
docker run -d -p 8080:80 expertuser/my_app:v1
```

> **🔥 자주 오해하거나 실수하는 부분:**
> `docker push`나 `docker pull` 시 **태그를 생략**하면 Docker는 자동으로 **`:latest` 태그**를 사용합니다. 따라서 버전 관리를 위해 태그를 명시하는 습관을 들이는 것이 중요합니다.

-----

Docker Hub는 CI/CD 파이프라인의 핵심이며, 이미지 등록 및 공유 방법을 숙지하는 것은 현대 IT 개발에서 필수적인 역량입니다.
